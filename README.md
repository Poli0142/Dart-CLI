# Магазин одежды - CLI система управления

## Выбранная предметная область
Интернет-магазин одежды с системой управления товарами, категориями, заказами и ролями пользователей.

## Структура папок

lib/
│ ├── domain/ # Бизнес-логика и модели
│ │ ├── models/ # Классы сущностей
│ │ ├── validators/ # Правила валидации
│ │ └── interfaces/ # Абстрактные классы
│ └── cli/ # Пользовательский интерфейс
│ ├── menu.dart
│ ├── input_helper.dart
│ └── display_helper.dart
├── data/ # Работа с БД
│ ├── database.dart
│ └── repositories/ # CRUD операции
├── bin/
│ └── main.dart # Точка входа
└── test/ # Тесты

### Domain 
- **Модели**: Содержат бизнес-сущности (Product, Category, Order, Role)
- **Валидаторы**: Чистые функции проверки данных
- **Почему**: Этот слой независим от внешних зависимостей, легко тестируется

### Data 
- **Database**: Управление SQLite соединением
- **Repositories**: CRUD операции для каждой сущности
- **Почему**: Изоляция работы с БД, легко заменить на другую СУБД

### CLI 
- **Menu**: Навигация и бизнес-логика
- **InputHelper**: Ввод с валидацией
- **DisplayHelper**: Форматированный вывод
- **Почему**: Отделяет пользовательский интерфейс от логики

## Валидации
### 1. Обязательное текстовое поле
- Проверка что строка не пустая после trim()
- Пример: название товара, имя покупателя
- Реализован в `RequiredStringValidator`

### 2. Числовое поле больше 0
- Проверка что цена, количество > 0
- Реализован в `PositiveNumberValidator`

### Дополнительные валидации
- Дата в формате ГГГГ-ММ-ДД
- Проверка наличия товара при заказе

## Связи между сущностями
- Product → Category (многие к одному)
- Order → Product (многие к одному)
- В БД реализованы FOREIGN KEY constraints
